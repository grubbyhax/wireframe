What is Wireframe?

Wireframe is a Cascading Stylesheet(CSS) utility built on top of all the marvelous work done in the Blueprint v0.4 release. Wireframe allows a web interface developer to quickly generate markup to suit the design specifications for any web development project.

Additional core functionallity was introduced to address issues of horizontal alignment and form typesetting. While, other selectors were dumped and/or modified according to issues of browser behaviour and personal taste.

Wireframe is a utility that produces two style sheets:

settings.css
Consolidated Blueprint framework file
layout.css
Markup document layout and styling file

Wireframe Assets

Here is a list of assests which are included in this release.

  • wireframe.html - User interface file

This is to give you some idea of the use of the classes included in the setting.css which Wireframe generates.

Wireframe Changelog

Because this utility is still in its infancy I'm not going to document every signle change I make. However, this system is already moving away from Blueprint. The very nature of how the grid works has been evolved to deal with the existing one's shortcomings in its treatment of graphic design assests.

The major improvement was to alter the mathematics involved in constructing the grid to allow for the successful transition to elastic layouts. Now, what was the old column, is defined as either a left or right floated screen. The append and prepend classes were changed to margins to allow for those creamy liquid centers.

How to Use Wireframe

The extra classes added to Blueprint in Wireframe allows a designer to use the grid to construct an interface with the classic screen/pane structure. All structural positioning is controled by a screen while, all interface alignment is done through the pane. This approach allows for the creation of perfectly typesetted forms with consistant page graphics.

I'm not going to waffle on about the screen/pane design approach or the ins and outs of using the classes in this system. Basically, you either understand cascading stylesheets and application interface development concepts, or you don't. This utiliy will not make you a better or more capable developer. It's intended goal is to speed up the development process.

The following is a description of the steps involved in the development cycle of an (X)HTML/CSS interface using Wireframe.

  1. Define typesettings
  2. Specify wieframes
  3. Output settings.css file
  4. Create grid image
  5. Develop markup code
  6. Add ids to structural elements
  7. Output layout.css file
  8. Consolidate additional css selectors in layout.css
  9. Comment out images in settings.css file

One Step at a Time

1. Define typesettings

The Wireframe grid is constructed dynamically using the Typesettings controls. The decision was taken to reduce the entire grid down to a single grid unit and then build the grid back up from there. The grid unit is referrenced directly to the font-size, which, is one and a half times the font size in pixels.

Font Size
Font size determines the size of the grid unit. The grid unit size is equal to the font size times one and a half.
Column Width
Column width defines the number of grid units wide each column is. Using the grid background, the columns are the blue vertical stripes.
Column Gutter
Column gutter defines the number of grid units wide each column gutter is. Using the grid background, the gutters are the white vertical stripes.
No. Columns
The number of columns defines the default width on the container class in columns, usually used to specify the page width. The container class has been deliberately kept low in the stylesheet hierarchy for easy override in specific use situations.

2. Specify wireframes

The Wireframes controls specify exactly how many additional stylesheet classes are added to position any pane <div> content. Obviously, some design concepts are going to require more positional classes. These controls will reduce code bloat for 'lite' positioning designs.

Max Grid Hang
Maximum amount of hang available on an element. Hang is an element's top padding in px.
Max Grid Lift
Maximum amount of lift available on an element. Lift is an element's bottom padding in px.
Max Grid Press
Maximum amount of press available on an element. Press is an element's bottom margin in px.
Max Grid Stretch
Maximum amount of stretch available on an element. Stretch is an element's height in em.

3. Output settings.css file

Once the typesetting and wireframes have been defined to your satisfaction, click on the Options radio selector and at the Settings CSS control choose Output. Your settings.css file will be generated and output into the CSS Code textarea Simply click the Select button to highlight the code and hotkey copy it. Then go to your favourite code editor, open up a new file and hotkey paste it in. Save the file as whatever name you like and you're ready to develop your document markup.

If you are unhappy with your settings, you can always change them and output a new settings.css file. If you want to start again from scratch you can simply click the Reset button for Settings CSS and off you go.

4. Create grid image

Before you do any markup development with the settings.css file, you are going to need a grid image to work with. So, open an image editor to work in. I recommend Microsoft Paint because it is small, fast and pixel gridded. Now, Open a new file to the size of one grid unit high and a width of one column width plus one gutter width. You will find these dimensions defined at the top of your settings.css file.

Fill the entire image in the colour you want your columns to be. Then, using a white brush, draw in the column gutter on the right hand side sides of the image the width of the gutter. Next, choose a new colour and draw the rule along the bottom of the image one pixel high. Save as a png file, or any other type you prefer, into your development folder.

Finally, you need to go back to the settings.css file and at the /* Images */ section, edit the image file path of the grid image you have created. Optionally, if you are doing wireframes as one stage in the development of your web project, you can add a frame image which, is a single pixel in the colour of the wireframe's bottom rule.

5. Develop markup code

Develop your markup code template as you normally would with vanilla Blueprint. Use any classes available in settings.css to control the positioning of elements within the page. With the extra wireframe classes you can position form elements more precisely and consistanly with greater attention to horizontal alignment.

6. Add ids to structural elements

All structural classes should now have an id attribute attached to define their semantic intention. Starting with the first element, copy its id and paste it into the Selector text input in the Identifiers control panel. Now copy the entire class definition of that element and paste it into the Classes input.

Now choose Enter.

That id, and the classes associated with it, are processed and held in the Wireframe document so, DO NOT REFRESH the page during the entirely of this process. In the Overflow text input, you will get output of any classes that are not involved in to core positioning class group. Paste those extra classes back into that structural element's class attribute.

Repeat this id/classes uploading process for each structural element in your markup.

7. Output layout.css file

When all the ids and associated classes have been uploaded, go to the Options control panel and click the Output button for Layout CSS. The layout.css file will be generated and outputted into the CSS Code textarea. Like the settings.css file, choose Select, then hotkey copy and paste into your favourite text editor and save as your desired file name.

One important point about using the two Wireframe stylesheets. They should ALWAYS be the first stylesheets to be imported into your page markup. And, in the order of settings.css first followed by layout.css.

8. Consolidate additional css selectors in layout.css

If you want, you can go back to the Identifiers control panel and use it to further remove markup class bloat by converting identical class combinations existing in elements to a single class that you can output and add to the layout.css file.

I made a conscious development decision not to have an either/or in the conversion algorithm between ids and classes. The basic reason why was the disruptive effects it can have on workflow and the stylesheet hierarchy. Arbitrary mixing of classes with ids has the potential for undesired browser display behaviour. Again, this utility does not teach you how to develop in CSS.

My recommendation is to do the classes separately as identifiers, output the code and in a separate document, do a find and replace of the id pointers(#) with the class pointers(.). Then, append this modified code to the existing pre-generated layout.css file. Below these additional classes, you can then proceed to add your own stylesheet declarations.

At the end of the day, it was quicker to have the utility work this way, than have a more sophisticated utility which would inevitably make the development process longer, more disorganised and painful.

9. Comment out images in settings.css file

The final step in the development cycle using Wireframe is to actually remove the grid images for the page. They have been isolated at the top of the setting.css file for your(...OK, my own) convenience under the /* Images */ comment. You can simply comment out those two lines of code and, voila, your grid will disappear.

All you need to do now is insert your graphic design assests into the page and you are ready to publish.

Some Q & A

1. Can I change Wireframe's default settings?

If you know only a little bit about javascript, you can go into the wireframe.js and wireframe.html files and edit the default settings to your own satisfaction, rather than always changing them from what I've defined as default.

You can find the default settings in the <form name="defaults"> element and in the UTILITY CONSTANTS section. It is important that you change these values in both places.

2. Can I change the default path of css images?

Yes, just goto the top of the wireframe.js file and you will find the image paths declared in the UTILITY CONSTANTS section. Simply replace the defaults with your own. Alternatively, you can simply replace the image file paths in the settings.css file after you have generated it.

3. How do I reincorporate Blueprint classes you ditched?

In the generateSettingsStylesheetCode() function in wireframe.js find the concantenating construction of sStylesheet and insert a new line at the point in which you want the stylesheet selector to appear. In fact, you can do this for any new selector you want to add to the settings.css output file. And, you can even add custom selectors to the layout.css file in the same way, placed outside the generateLayoutStylesheetCode() function's main loop.

4. Can I add new classes into identifier selectors?

Yes you can, and it's a reasonably straight forward process. The first step is to locate the identfiers table, <table id="identifiers">, in the wireframe.html file. What you need to do is add a new table row, preferrably at the end, of the class name you are going to define in the settings.css file. If the new class is a stepped value classname-# then, make sure you are only inserting the actual class name into this table and not the suffix demarkation.

Next, you need for the utility to output the value of your new classes in both the settings.css and layout.css files. The generateSettingsStylesheetCode() function generates the settings.css so you simply insert the line of code for your new class into it. If the class is stepped then you will have to create a new function to insert into the line of code you want to output it on. For the novice javascripters out there, there are enough of these functions used for you to be able to template your own.

To include your new class in the layout.css file, you need to modify the generateLayoutStylesheetCode() function. To do this, you will need to increment the nNumOfClasses variable by one per additional class and then create an new matching if statement for the class you want to insert. The array holding the markup properties is aStyleValues and each position holds the following properties:

  • aStyleValues[0] - position
  • aStyleValues[1] - float
  • aStyleValues[2] - width
  • aStyleValues[3] - height
  • aStyleValues[4] - margin-top
  • aStyleValues[5] - margin-right
  • aStyleValues[6] - margin-bottom
  • aStyleValues[7] - margin-left
  • aStyleValues[8] - padding-top
  • aStyleValues[9] - padding-right
  • aStyleValues[10] - padding-bottom
  • aStyleValues[11] - padding-left
  • aStyleValues[12] - text-align
  • aStyleValues[13] - clear

If you are going to add a new style property then add it to a new array index and don't forget to include that property in the construction of the sStylesheet string.

One important thing to note, all the properties except for margin and padding are assigned as full strings on identification match. Whereas, margin and padding are shortened and re-processed near the end of the main function loop to ensure the CSS output is correct. So, watch out for this logic if you are going to add classes with margin and padding properties.
